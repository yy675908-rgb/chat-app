# 林间

Flutter / Android 优先的本地角色聊天 App。核心目标不是做一个普通问答框，而是让角色拥有连续的关系、记忆和状态。

## 产品边界

角色不是每次打开后才临时出现的回答机器。它应记得共同经历、感知时间经过，并逐渐形成连续的关系。

## 当前功能

- 内置角色“林”，可添加、切换和独立设置多个角色
- 支持多个供应商与模型，兼容 OpenAI Chat Completions 和 Anthropic Messages API
- 聊天页可在已添加的供应商与模型之间随时切换
- 流式显示正文和接口原生思考内容，并记录思考用时
- 回复可重新生成多个版本；每个版本拥有独立的后续对话分支
- 多会话抽屉、自动命名、手动改名、停止生成与连续发送
- 长对话可提炼为分支摘要，原始消息仍完整保留
- 共同记忆、回应偏好、世界书、收藏和角色心绪
- Markdown 回复、代码块、复制、token 用量
- 柔白浅色界面，使用低饱和蓝灰交互色，避免刺眼纯白和米黄色调
- 聊天记录、角色档案、供应商配置和记忆保存在本机，可导出/导入 JSON 备份
- API Key 使用 Android 加密存储

调用模型时，必要的聊天上下文会发送给用户自行配置的 API 供应商；App 本身不设服务器。

## 独立实现

本项目独立实现，未使用其他项目的源码或视觉资产，未改造其他成品代码。角色关系、分支对话、心绪、记忆与世界书交互均为独立设计与实现。

## 后续方向

1. 将大量聊天记录迁移至 SQLite，并增加全文搜索。
2. 增加角色导入导出和可选的角色语音。
3. 增加可关闭的本地通知与主动消息。

每次推送到 `main` 后，GitHub Actions 会自动生成 Android 工程、执行分析与测试，并编译 release APK。也可在本地安装 Flutter 后运行：

```bash
flutter create . --platforms=android
flutter pub get
flutter run
```
